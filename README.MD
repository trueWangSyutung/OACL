## Introduction 
With the deepening development of open science, the free flow of academic knowledge is crucial to human progress. This license is specifically designed for the open source requirements of academic paper code, adhering to the Ultra Open Access principle to ensure the complete openness, immediate availability, and collaborative friendliness of academic code. This license emphasizes the core values ​​of the academic community, eliminates unnecessary barriers to knowledge dissemination, and protects the legitimate rights of authors.

## Chapter 1: Definitions

### Original Paper 
Code Refers to the original software code, algorithm implementations, data processing scripts, and other digital content described and published alongside academic papers, including but not limited to research source code, simulation models, and data analysis scripts.

### Open-Source Derivative Code 
Code versions modified, optimized, extended, or refactored based on original paper code must also be released in an open-source manner.

### Author

The creator or copyright holder of the original paper's code.

### Licensee

Any individual or organization that uses, modifies, or distributes the paper's code under this agreement.

### Non-Commercial Use

Use not for direct or indirect economic gain, including: academic research, education, personal learning, non-profit projects, public academic exchange, etc.

### Commercial Use

Use for the purpose of generating economic gain, including: commercial product sales, paid licensing, advertising-funded distribution, internal R&D applications, code-based paid services, etc.

### Dataset-Easy-Access Principle

The dataset associated with the paper should be easily accessible, with simplified licensing procedures, while respecting data privacy and confidentiality requirements.

## Chapter 2: Core Terms

### 2.1 Ultra Open Access Obligation

Mandatory Open Source Requirement
All code used in papers under this license must be released as open source, without any form of closed-source restriction.

Timeliness of Release
When the paper is officially published, the relevant code should be simultaneously or in advance made public on a recognized code hosting platform (such as GitHub, GitLab, OSF, etc.).

Code Integrity
The released code should contain the necessary files sufficient to reproduce the core results of the paper, including but not limited to: source code, dependency descriptions, running examples, and documentation comments.

### 2.2 Non-Commercial Use Terms

#### Free Use License
In non-commercial use cases, users may use the following for free:

* Download, copy, and run the code

* Modify and create derivative versions

* Distribute copies of the code in academic exchanges

* Collaborate across institutions

#### Restrictions on For-Profit Use
Any for-profit use must simultaneously meet the following conditions:

* Obtain the author's explicit written permission

* Reach a fair and reasonable compensation agreement with the author

* Enter into a clear legal usage contract

### 2.3 Non-Policy Principle
This agreement and the code distributed under this agreement do not carry any specific political stance and may not be forced to be used for:

* Propaganda of specific political ideologies

* Political campaigns or lobbying activities

* Censorship tools that restrict academic freedom

* However, it is permitted for academic analysis researching politically relevant topics.

### 2.4 Allow-Non-Institution Flexibility

**Author affiliation identification is optional; authors can choose whether to add an institution/country identifier.**

* Authors can choose whether to add their affiliation information when publishing code. The absence of an institution identifier does not affect the validity of the agreement.

Attribution Requirements

* The original author's name and contact information (such as ORCID, email, etc.) must be retained in the code to ensure traceability of academic contributions.

* No country/region needs to be indicated in the paper.

### 2.5 Dataset-Easy-Access Terms

#### Simplified Data Authorization
Datasets associated with the paper should use a simplified authorization process. Recommended protocols include:

* Creative Commons licenses (CC series)

* Domain-specific data sharing licenses

* Data sharing appendix to this license

#### Privacy and Confidentiality Protection
If the dataset contains sensitive or privacy information:

* A de-identified version or mock dataset must be provided.

* A data usage agreement can be set to protect the rights of the parties involved.

* Access to the original data should be allowed through controlled access.

#### Data Documentation Integrity

* The dataset should be accompanied by a clear data dictionary, data collection methods, and usage restrictions.
